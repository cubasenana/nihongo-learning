# 开课前大纲审查 Gate

当用户要求开始 lesson-X 教学时，教学 AI 不得立即讲课。必须先执行本 Gate，只审查、不教学。

Gate 的任务是判断本课材料是否可以安全进入教学；发现 P0/P1 时必须阻断教学，不得“边讲边修”。

审查停止规则：

- P0 若导致文件无法读取或结构无法解析，立即停止审查并报告。
- P0 若不影响继续读取，继续审完并列入阻塞项。
- P1 不停止本轮审查，只停止后续教学；应尽量一次性列全本课 P1 问题。

## 问题优先级定义

| 级别 | 定义 | 处理 |
|------|------|------|
| P0 | 文件缺失、栏目缺失、乱码、Markdown 表格损坏等导致材料无法正常读取 | 停止教学，报告问题 |
| P1 | 语法越界、核心语法未覆盖、覆盖清单和正文矛盾、日语硬错误、敬语对象错误、人物/场景硬矛盾 | 停止教学，报告问题 |
| P2 | 日语略不自然、覆盖偏弱、`📖` 标记不一致、表达要点与本课关联弱 | 可以教学，但必须在教学时提醒 |
| P3 | CSV 词源不精确匹配、生词表可补充、表达优化建议 | 不影响教学 |

## Step 1：读取边界

读取：

- `docs/syllabus/lesson-XX.md`
- `docs/syllabus/grammar-unlock.md`
- `resources/jlpt-vocab-n5n4.csv`

输出：

- 本课新增语法
- lesson <= X 可用语法
- 本课生词表中的核心 `📖` 词
- 本课练习阶段类型：L01-L04 / L05-L08 / L09+

若任一必要文件不存在，标记 P0。

## Step 2：结构检查

检查：

- 是否有完整栏目：`## 语法点`、`## 课文`、`## 生词表`、`## 表达要点`、`## 练习框架`
- Markdown 表格列数是否一致（表头列数 = 数据行列数）
- 是否有乱码 `�`（U+FFFD replacement character）
- 课文和语法覆盖清单是否存在
- 生词表是否有 `📖` 标记

发现 P0 时按“审查停止规则”处理。

## Step 3：语法边界检查

检查：

- 课文是否使用 lesson > X 的语法
- 练习框架是否要求 lesson > X 的语法
- 表达要点是否提前引入后课语法
- 本课每个 `###` 语法点是否在课文中有真实例句
- 语法覆盖清单是否和正文一致

语法覆盖判定规则：

- 具体句型、接续、活用形必须在课文中有真实例句。
- `敬语概述`、`自谦语概述`、分类说明、文化说明等概念型条目，可以通过同课相关例句整体覆盖，不强制要求独立例句。
- 若一个 `###` 标题包含多个并列形式（如 `よう/みたい`、`間/間に`），原则上每个形式都要有例句；若本课大纲明确只要求识别其中一个，需在 Gate 结果中说明。

固定表达例外：

- 寒暄、店铺常用语、商务套话中含有后课语法时，可以作为固定表达识别。
- 固定表达例外不得用于本课主动造句要求。
- 若固定表达会误导 0 基础学生理解本课语法，标记 P2 或 P1。

发现 P1 时继续完成本轮审查，但最终必须阻断教学。

## Step 4：Bunpro 校验

对本课每个语法点做一次联网校验。

Bunpro 条目匹配规则：

1. 先从本地 `###` 标题中抽取日语核心 pattern，去掉中文说明、编号、括号补充和本项目内部标签。
2. 一个标题含多个 pattern 时，拆开分别校验，例如 `よう/みたい`、`ていきます/てきました`。
3. 优先搜索 `site:bunpro.jp/grammar_points {pattern}`，或访问公开的 Bunpro grammar point 页面。
4. 多个候选都可能匹配时，在 `Bunpro 校验摘要` 中列出候选，并说明采用哪一个。
5. 找不到对应条目时，标记 `无对应条目` 或 `未能确认对应条目`，不得硬套相近条目。
6. 概念型条目（如敬语概述、分类总览）允许标记 `无单独 Bunpro 条目`，不因缺少条目自动判为 P1。

校验内容：

1. 对比本地大纲中的含义、接续、使用限制、例句方向。
2. 若本地讲解与 Bunpro 冲突：
   - 影响语法正确性或接续规则：标记 P1。
   - 只影响补充说明或自然度：标记 P2。

无法访问 Bunpro 时，不得假装校验完成。记录：

```text
Bunpro：未能校验，本课讲解仅基于本地大纲和 AI 判断。
```

Bunpro 访问失败处理：

- 单个语法点无法校验：标记 P2，教学时必须提示。
- 整课 Bunpro 都无法访问：仍可继续 Gate 的本地审查，但教学许可最多只能是 `可以教学，但有 P2/P3 注意事项`。
- 若本课语法高风险（敬语、使役、被动、授受、条件限制）且 Bunpro 未校验，同时本地解释存在疑点，标记 P1。

Gate 已完成的 Bunpro 校验结果必须传给 `teaching-flow.md` 复用。正式教学阶段不重复完整联网校验，除非学生追问某个语法点或 Gate 结果缺失。

## Step 5：日语自然度检查

检查：

- 助词使用
- 时态一致性
- 敬体/简体是否符合场景
- 尊他/自谦对象是否正确
- 对话角色逻辑（说话人和听话人身份）
- 叙事连续性（与前后课故事线）
- 0 基础学生是否能按当前进度理解

处理：

- 敬语对象错误、说话人自问自答、被动/使役对象反转等会误教学生的错误，标记 P1。
- 略不自然但语法正确且不影响教学目标的表达，标记 P2。

## Step 6：词汇检查

检查：

- 课文中出现的重点词是否在生词表中
- `📖` 标记是否和课文实际出现一致，并区分核心词与场景例外词
- 本课新增词是否来自 `resources/jlpt-vocab-n5n4.csv`
- 非 CSV 词是否属于允许例外

切词和归并规则：

- 只检查内容词和固定表达：名词、动词、形容词、副词、连体词、寒暄/商务套话。
- 不把助词、助动词、活用尾、接续助词、普通标点、纯数字单独列为缺词。
- 动词按辞书形归并：`買いました/買って/買える` 归到 `買う` 或本课实际学习形。
- い形容词按辞书形归并：`高かった/高くて` 归到 `高い`。
- な形容词和名词去掉 `です/だ/だった` 后归并。
- 复合词优先按课文教学需要处理；若拆分后学生仍无法理解，应把复合词列为词表候选。
- 专名、场景词和固定表达按“允许例外”处理。

允许例外：

- 人名、地名、公司名、作品名、店名等专有名词
- 商务电话、寒暄、接待等固定套话
- 场景成立所必需的少量主题词

例外词处理：

- 可以出现在课文中。
- 可以列入生词表，但应标明其场景性或固定表达属性。
- 不应作为进入下一课的核心词汇门槛，除非课文理解离不开。

## Step 7：给出教学许可

必须按以下格式输出 Gate 结果。

### Gate Result

结论：`可以教学` / `可以教学，但有 P2/P3 注意事项` / `不能教学，需先修 P0/P1`

### P0/P1 阻塞项

| 级别 | 文件位置 | 问题 | 为什么阻塞 | 本课范围内修正建议 |
|------|----------|------|------------|--------------------|

若无阻塞项，写：`无`。

### P2/P3 非阻塞项

| 级别 | 文件位置 | 问题 | 教学时如何处理 |
|------|----------|------|----------------|

若无非阻塞项，写：`无`。

### Bunpro 校验摘要

| 语法点 | 采用条目/URL | Bunpro 状态 | 是否一致 | 注意事项 |
|--------|-------------|-------------|----------|----------|

### Lesson Record 种子

若允许教学，预先建立本课记录框架，正式教学结束时由 `teaching-flow.md` 填写：

| 项目 | 内容 |
|------|------|
| 本课 P2/P3 注意事项 | |
| 需复查语法 | |
| 需复查词汇 | |

### 教学许可

- 无 P0/P1：进入 `md/prompts/teaching-flow.md`。
- 有 P0/P1：停止，不讲课，要求先修材料。
