# L45-L48 课文与生词表审核报告

审核对象：`docs/syllabus/lesson-45.md` 到 `docs/syllabus/lesson-48.md` 的新增 `## 课文` 与 `## 生词表`。

审核基准：

- 语法解锁：`docs/syllabus/grammar-unlock.md`
- 只检查 `lesson <= X` 的语法是否越界
- 对照每课 `## 语法点` 下的 `###` 标题检查覆盖
- 同时检查日语自然度、叙事连续性、生词表 `📖` 标记、表格列数、词源一致性

## 总结

最后四课整体可用，但需要修的点比较集中：

- L46 `カフェがあるみたいです` 是 L44 的推测用法，不是 L46 的比喻用法，所以 `みたいです（比喻）` 覆盖误判。
- L47 课文没有真正出现 `～（ら）れます` 尊他语，覆盖清单里的 `集まられた` 不在课文中。
- L47 `教えていただきました` 严格按解锁表看，提前用了 L48 自谦/受益表达。
- L48 课文没有出现 `～ていただけますか`，但使用语法和覆盖要求里都写了它。
- L47、L48 的特殊敬语表格标题少了 `📖` 列，Markdown 表格列数不一致。

Markdown 表格列数检查：L45-L46 通过；L47、L48 的特殊形式表存在列数错误。

## L45

### ✅ 语法越界检查

未发现明确后课语法越界。`てきました` 变化/出现、`ていきます`、`し`、`ば～ほど`、`助词＋の` 都属于 L45；`～について` 已在 L24 表达中出现，可视为已学。

### ✅ 语法覆盖

L45 五个语法点实际覆盖完整：

- `てきました` 变化：`変わってきました`、`増えてきました`、`進めてきました`
- `ていきます`：`集めていきます`
- `てきました` 出现：`おなかがすいてきたり`、`疲れてきたり`
- `し`：`便利になりましたし`、`増えましたし`
- `ば～ほど`：`住めば住むほど`
- `助词＋の`：`上海への出張`、`上海までの距離`

### 💡 覆盖清单小漏写

位置：`lesson-45.md:108`

覆盖清单只写了：

```text
てきました変化（変わってきた/増えてきた/進めてきた）
```

但课文其实也有：

```text
来月からは 本格的に スタッフを 集めて いきます。
```

建议把覆盖清单补成：

```text
ていきます（集めていきます） / てきました変化（変わってきました/増えてきました/進めてきました）
```

### ⚠ 自然度微调：`上海への出張は 3時間ぐらいです`

位置：`lesson-45.md:104`

原文：

```text
東京から 上海への 出張は 3時間ぐらいです。
```

意思能懂，但 `出張は3時間` 容易被理解成“出差总共三小时”。如果想表达路程/飞行时间，更自然的是：

```text
東京から 上海までは 飛行機で 3時間ぐらいです。
```

## L46

### ✅ 语法越界检查

未发现明确后课语法越界。`あるみたいです` 属于 L44 已学的推测用法，不算越界；但它不能覆盖本课的比喻用法。

### ❌ 语法覆盖误判：`みたいです（比喻）` 未覆盖

位置：`lesson-46.md:107`, `lesson-46.md:117`

原文：

```text
王：あ、隣に カフェが あるみたいです。
```

覆盖清单写：

```text
みたいです比喩（カフェがあるみたい）
```

问题：

这里的 `みたいです` 是“好像有咖啡馆”的推测，不是“像……一样”的比喻。L46 的 `よう/みたい` 重点是比喻，应补一个真正的比喻句。

建议替换或补一句：

```text
王：この 部屋は 図書館みたいに 静かですね。
```

并把覆盖清单改成：

```text
みたいです比喩（図書館みたいに静か）
```

### ✅ 其他语法覆盖

其余 L46 语法点覆盖正常：

- `ようです` 比喻：`ホテルのロビーのよう`
- `らしい` 典型性：`上海らしい雰囲気`
- `までに`：`月末までに`、`来月の初めまでには`
- `間/間に`：`見学している間`、`見学している間に`
- `がします`：`コーヒーのような匂いがします`

### ✅ 日语自然度

除 `みたいです` 用法与教学目标不符外，对话整体自然。`見学している間に、いくつか気づいたことがあります`、`見学している間、ずっとこの匂いがしていました` 对 `間/間に` 的区分清楚。

### ✅ 叙事连续性

L46 事务所选址承接 L45 上海筹备线，没有发现人物、地点、时间线矛盾。

### 💡 生词表/词源一致性

`📖` 标记与课文实际用词基本一致。

若严格执行 CSV 词源，以下 L46 表内词条未能在 CSV 中精确匹配到词条：

- `ロビー`
- `雰囲気`
- `契約`
- `済ませます`
- `カフェ`

## L47

### ❌ 语法覆盖缺失：`～（ら）れます` 尊他语未出现

位置：`lesson-47.md:105`, `lesson-47.md:123`

原文：

```text
社長：きれいだね。もう スタッフは 集まったか。
```

覆盖清单写：

```text
(ら)れます尊他（集まったか→集まられた — 社長は簡体使用）
```

问题：

课文中实际出现的是社长对下属说的普通简体 `集まったか`，不是尊他语。`集まられた` 只是覆盖清单里的解释，并没有进入课文，所以不能算覆盖 `～（ら）れます`。

建议补一句真正的尊他被动形，例如：

```text
陳：社長、昨日は よく 休まれましたか。
```

或：

```text
陳：こちらの 資料は もう 読まれましたか。
```

### ❌ 严格语法越界：`教えていただきました`

位置：`lesson-47.md:115`

原文：

```text
陳：ありがとうございます。皆様に いろいろ 教えて いただきました。
```

问题：

`いただきました` 是 `いただく`，属于自谦/受益表达。按 `grammar-unlock.md`，自谦语系统在 L48 解锁；L47 的重点是尊他语。严格按 `lesson <= 47`，这一句提前用了 L48 内容。

建议改成 L47 已教的尊他特殊形式 `くださいます`：

```text
陳：ありがとうございます。皆様が いろいろ 教えて くださいました。
```

这样还能补到特殊尊他 `くださいます`。

### ⚠ 自然度：感谢对方到访时不宜用 `お越しになりまして`

位置：`lesson-47.md:99`

原文：

```text
陳：社長、お忙しい 中、上海まで お越しに なりまして、ありがとうございます。
```

问题：

`お越しになります` 本身是正确的尊他语，但放在“感谢对方到访”的固定表达里，日语更常用 `お越しくださり / お越しくださいまして`。

建议：

```text
陳：社長、お忙しい 中、上海まで お越しくださり、ありがとうございます。
```

或：

```text
陳：社長、お忙しい 中、上海まで お越しくださいまして、ありがとうございます。
```

### ❌ 生词表格式：特殊尊他表格列数不一致


## L48

### ✅ 语法越界检查

L48 是最后一课，课文中的自谦语、礼貌语、商务电话表达均在本课或前课范围内，未发现后课越界问题。

### ❌ 语法覆盖缺失：`～ていただけますか` 未出现

位置：`lesson-48.md:112`, `lesson-48.md:140`

使用语法写了：

```text
ていただけますか
```

但课文正文没有任何 `～ていただけますか / ～ていただけませんか`。覆盖清单也没有列出对应例句。

建议补一句商务电话中很自然的请求：

```text
陳：すみませんが、来週の ご都合を 教えて いただけますか。
```

如果想避开 `ご都合` 这个词，也可以更基础：

```text
陳：すみませんが、来週の 時間を 教えて いただけますか。
```

### ⚠ 覆盖偏弱：`ご～します` 变体未出现

位置：`lesson-48.md:140`

`お/ご～します` 这个语法点中，课文有：

```text
日程を お知らせします。
午後2時に お待ちして おります。
```

这能覆盖 `お～します`，但没有 `ご～します`。如果希望把两个形态都给 0 基础学生看一遍，建议补一句：

```text
陳：後ほど、メールで ご連絡します。
```

### ✅ 特殊自谦与 `ございます`

- `ございます`：`JC企画 上海事務所でございます`

- `申します`：`山本と申します`、`陳と申します`
- `拝見いたしました`：`ご案内を拝見いたしました`
- `伺います`：`そちらに伺いたい`
- `参ります`：`午後なら参れます`
- `させていただきます`：`お送りさせていただきます`

## 建议修复顺序

1. 先修 L48，补 `～ていただけますか`，这是最后一课最明显的覆盖缺失。
2. 再修 L47，补真正的 `～（ら）れます` 尊他句，并把 `教えていただきました` 改成 `教えてくださいました`。
3. 修 L46，把 `カフェがあるみたいです` 从“比喻覆盖”中拿掉，补一个真正的 `みたい` 比喻句。
4. 修 L47/L48 特殊形式表格表头，补上 `📖` 列，避免 VitePress 表格渲染错位。
5. 最后按是否严格执行 CSV 词源，统一处理 L45-L48 未精确匹配 CSV 的词条。
